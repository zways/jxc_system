## 一、快速入门

### 1.1 企业注册

首次使用系统需要进行企业注册，注册完成后即可开始使用。

**注册入口：** 在系统登录页面，点击底部「**立即注册**」链接即可进入注册页面。

**注册步骤：**

1. 打开系统登录页 → 点击「立即注册」
2. 填写**企业信息**：
   - **企业名称**（必填）：您的公司或门店名称
   - **所属行业**（选填）：从下拉菜单选择（零售、批发、制造、食品饮料等）
   - **营业执照号**（选填）
   - **企业地址**（选填）
3. 填写**管理员账号**：
   - **管理员姓名**（必填）：管理员真实姓名
   - **邮箱**（必填）：用于登录系统的邮箱地址
   - **手机号**（选填）
   - **登录密码**（必填）：至少6位
   - **确认密码**（必填）：再次输入密码
4. 点击「立即注册」

**注册完成后系统自动创建：**
- 您的企业账号
- 管理员角色（拥有全部权限）
- 默认部门（管理部）
- 默认仓库（编码：WH001）
- 管理员用户账号

> 💡 **提示：** 注册成功后系统会自动登录并跳转到首页仪表盘，无需再次手动登录。请妥善保管您的邮箱和密码。

---

### 1.2 登录系统

1. 打开系统登录页面
2. 输入 **用户名**（或邮箱）和 **密码**
3. 点击「登录」按钮
4. 登录成功后自动跳转到首页仪表盘

**忘记密码：** 登录页提供「忘记密码？」链接。点击后输入注册时使用的**邮箱**，系统会发送重置链接到该邮箱；按邮件中的链接设置新密码即可。若管理员已关闭找回密码功能，或无法收到邮件，请联系企业管理员在「用户管理」中为您重置密码。

> ⚠️ **注意：** 系统支持多个用户同时在线使用，每个用户只能看到自己权限范围内的数据。

---

### 1.3 首页仪表盘

登录后首先看到的是**仪表盘**页面，这里集中展示企业运营关键数据：

**核心指标卡片：**
| 指标 | 说明 |
|------|------|
| 总销售额 | 本月销售总额及同比增长率 |
| 订单数量 | 本月订单总数 |
| 库存总额 | 当前库存总价值及商品种类数 |
| 应收账款 | 待收款总金额 |

**功能区域：**
- **销售趋势图** — 展示销售额变化趋势
- **库存状况图** — 展示库存分布情况
- **快捷操作** — 快速跳转到常用功能（新建销售单、新建采购单、库存盘点等）
- **待办事项** — 显示待处理的采购单和销售单
- **日期范围选择** — 可切换查看不同时间段的数据

## 二、系统初始化设置（首次使用必读）

> 🔔 **重要：** 首次使用系统，建议按照以下顺序完成基础数据配置。基础数据是所有业务操作的前提。

### 2.1 设置流程概览

```
第1步：部门管理 → 设置公司组织架构
第2步：仓库管理 → 创建实际仓库
第3步：门店管理 → 创建门店/分支（如有）
第4步：角色与权限 → 定义不同岗位的操作权限
第5步：用户管理 → 为员工创建账号
第6步：计量单位 → 设置商品计量单位（个、箱、kg 等）
第7步：商品分类 → 建立商品分类体系
第8步：商品管理 → 录入商品信息
第9步：供应商管理 → 录入供应商信息
第10步：客户管理 → 录入客户信息
第11步：业务员管理 → 录入业务员信息（可选）
```

---

### 2.2 部门管理

**路径：** 系统设置 → 部门管理

用于管理公司的组织架构。注册时系统已自动创建「管理部」。

**操作说明：**

| 操作 | 说明 |
|------|------|
| 新增部门 | 点击「新增部门」按钮，填写部门名称、部门编码、描述 |
| 编辑部门 | 点击对应行的「编辑」按钮修改信息 |
| 删除部门 | 点击对应行的「删除」按钮（部门下有用户时不可删除） |

**字段说明：**
- **部门名称**（必填）：如「销售部」「采购部」「仓储部」「财务部」
- **部门编码**（必填）：唯一标识，如 `SALES`、`PURCHASE`、`WAREHOUSE`
- **描述**（选填）：部门职责说明

---

### 2.3 仓库管理

**路径：** 系统设置 → 仓库管理

管理企业的实体仓库。注册时系统已自动创建默认仓库（WH001）。

**操作说明：**

| 操作 | 说明 |
|------|------|
| 新增仓库 | 点击「新增仓库」按钮，填写仓库信息 |
| 编辑仓库 | 点击对应行的「编辑」按钮修改信息 |
| 删除仓库 | 点击对应行的「删除」按钮（有库存记录时请谨慎操作） |

**字段说明：**
- **仓库编码**（必填）：唯一标识，如 `WH001`、`WH002`
- **仓库名称**（必填）：如「主仓库」「华东仓」
- **类型**（必选）：常规 / 冷链 / 液体
- **管理员**（选填）：仓库负责人
- **位置**（选填）：仓库地址
- **状态**：启用 / 禁用

> 💡 **建议：** 根据实际仓库数量创建，不建议创建过多虚拟仓库。

---

### 2.4 门店管理

**路径：** 系统设置 → 门店管理

如果您的企业有多个门店或分支机构，可在此管理。

**字段说明：**
- **门店编码**（选填，可自动生成）
- **门店名称**（必填）：如「总店」「城东分店」
- **类型**（必选）：零售 / 批发 / 线上 / 混合
- **上级门店**（选填）：支持多级门店结构
- **负责人 / 电话 / 地址**（选填）
- **状态**：启用 / 禁用

> ⚠️ **注意：** 存在下级门店时，上级门店不可删除。

---

### 2.5 角色与权限配置

**路径：** 系统设置 → 角色配置

系统采用基于角色的权限控制（RBAC），您可以为不同岗位创建不同的角色。

**第一步：创建角色**

| 字段 | 说明 | 示例 |
|------|------|------|
| 角色名称 | 角色的中文名称 | 采购员、销售员、仓管员、财务主管 |
| 角色编码 | 角色的唯一标识 | purchaser、salesperson、warehouse_keeper |
| 描述 | 角色职责说明 | 负责日常采购业务 |

**第二步：分配权限**

1. 在角色列表中点击「权限设置」按钮
2. 系统展示所有权限的树形结构，按模块分组：

| 权限分组 | 包含权限 |
|----------|----------|
| 系统管理 | 用户管理、角色管理、部门管理、仓库管理、门店管理等 |
| 采购管理 | 供应商管理、采购订单管理、入库记录管理 |
| 销售管理 | 客户管理、销售订单管理、出库记录管理、退货/换货管理 |
| 库存管理 | 库存查询、库存调整、库存盘点、调拨管理 |
| 财务管理 | 应收账款、应付账款、收支明细 |
| 报表中心 | 各类报表查看权限 |

3. 勾选该角色需要的权限
4. 点击「保存」

**推荐角色配置示例：**

| 角色 | 建议权限范围 |
|------|-------------|
| 管理员 | 全部权限 |
| 采购员 | 采购管理全部 + 供应商管理 + 库存查看 |
| 销售员 | 销售管理全部 + 客户管理 + 库存查看 |
| 仓管员 | 库存管理全部 + 采购入库查看 + 销售出库查看 |
| 财务 | 财务管理全部 + 报表中心 |
| 查看者 | 仅各模块的查看权限 |

### 2.6 用户管理

**路径：** 系统设置 → 用户管理

**字段说明：**
- **用户名**（必填）：3-20 个字符，登录时使用
- **真实姓名**（必填）：员工真实姓名
- **密码**（必填）：6-20 个字符
- **确认密码**（必填）：与密码一致
- **邮箱**（必填）：有效邮箱地址
- **电话**（必填）：手机号码
- **角色**（必选）：从已创建的角色中选择
- **部门**（必选）：从已创建的部门中选择
- **工号**（选填）：员工工号
- **状态**：启用 / 禁用

> 💡 **提示：** 禁用用户后，该用户将无法登录系统但数据保留。

## 三、采购管理

### 3.1 采购流程概览

```
创建采购订单 → 确认收货 → 自动生成入库记录 + 应付账款 → 付款 → 完成
     ↓                                                          ↑
   (可取消)                                              (在财务模块操作)
```

**订单状态流转：**
| 状态 | 说明 |
|------|------|
| 待处理 (pending) | 刚创建，等待确认 |
| 已确认 (confirmed) | 已审核确认 |
| 已收货 (received) | 货物已到并确认入库 |
| 已取消 (cancelled) | 订单已作废 |

---

### 3.2 创建采购订单

**路径：** 采购管理 → 采购订单 → 点击「新增采购单」

**操作步骤：**
1. **选择供应商**（必选）：从供应商列表中选择
2. **选择仓库**（必选）：货物将入库到此仓库
3. **填写订单日期**（必填）
4. **添加采购明细**：
   - 点击「添加明细」或使用**扫码添加商品**输入框（支持 PDA 扫码枪，扫码后自动添加并累加数量）
   - 选择商品
   - 填写采购数量

## 四、销售管理

### 4.1 销售流程概览

```
创建销售订单 → 确认发货 → 自动生成出库记录 + 应收账款 → 收款 → 完成
     ↓                                                          ↑
   (可取消)                                              (在财务模块操作)
     
如有退货：
   销售退货 → 退货处理(入库+冲减应收) → 退款处理(如有超收)
     
如有换货：
   换货处理 → 退回入库 + 换出出库
```

**订单状态流转：**
| 状态 | 说明 |
|------|------|
| 待处理 (pending) | 刚创建，等待确认 |
| 已确认 (confirmed) | 已审核确认 |
| 已发货 (delivered) | 货物已发出 |
| 已取消 (cancelled) | 订单已作废 |

**订单类型：**
| 类型 | 说明 |
|------|------|
| 零售 (retail) | 零售订单 |
| 批发 (wholesale) | 批发订单 |
| POS (pos) | POS 终端订单 |

---

### 4.2 创建销售订单

**路径：** 销售管理 → 销售订单 → 点击「新增销售单」

**操作步骤：**
1. **选择客户**（必选）：从客户列表中选择
2. **选择仓库**（必选）：货物将从此仓库出库
3. **填写订单日期**（必填）
4. **选择订单类型**：零售 / 批发 / POS
5. **添加销售明细**：
   - 点击「添加明细」或使用**扫码添加商品**输入框（支持 PDA 扫码枪，扫码后自动添加并累加数量）
   - 选择商品
   - 填写销售数量
   - 填写销售单价（默认读取零售价或批发价）
   - 系统自动计算金额 = 数量 × 单价
6. **填写备注**（选填）
7. 点击「确定」保存订单

> 💡 **提示：** 订单编号由系统自动生成，格式为 `SO + 日期 + 序号`，如 `SO202602100001`。

> ⚠️ **重要：** 发货前请确保仓库有足够库存，否则系统会提示库存不足并阻止发货。

### 4.4 取消销售订单

## 五、库存管理

### 5.1 实时库存查询

**路径：** 库存管理 → 实时库存

查看所有商品的当前库存数量（只读页面）。

**展示信息：**
- 商品编码、商品名称
- 所属分类
- 计量单位
- 最低库存 / 最高库存
- **当前库存**（系统实时计算）
- 库存状态（正常 / 库存不足 / 库存过多）

**筛选功能：**
- 按商品名称或编码搜索
- 按仓库筛选

**库存计算规则：**

| 业务类型 | 对库存的影响 |
|----------|-------------|
| 入库 (in) | +（增加） |
| 出库 (out) | -（减少） |
| 调整 (adjust) | +或-（根据调整方向） |
| 调拨 (transfer) | 调出仓-、调入仓+ |

> 💡 **提示：** 库存数据是通过汇总所有库存流水记录实时计算得出的，确保数据准确。

---

### 5.2 库存调拨

**路径：** 库存管理 → 调拨管理

在不同仓库之间转移库存。

**操作步骤：**
1. 点击「新增调拨单」按钮
2. 选择商品（可通过**扫码输入框**扫码或输入条码/编码后按回车快速选择）
3. 选择**调出仓库**
4. 选择**调入仓库**（不能与调出仓库相同）
5. 填写调拨数量
6. 填写单位成本
7. 添加备注（选填）
8. 点击「确定」提交

**系统自动完成：**
- ✅ 在调出仓库生成一条「调拨出库」记录
- ✅ 在调入仓库生成一条「调拨入库」记录
- ✅ 调出仓库库存减少，调入仓库库存增加

---

### 5.3 库存调整

**路径：** 库存管理 → 库存调整

用于手动调整库存数量，适用于以下场景：
- 初始库存录入
- 损耗报废
- 赠品出入库
- 其他需要手动调整的情况

**操作步骤：**
1. 点击「新增调整」按钮
2. 选择商品（可通过**扫码输入框**扫码或输入条码/编码后按回车快速选择）
3. 选择仓库
4. 选择类型：**增加** 或 **减少**
5. 填写调整数量（最小 0.01）
6. 填写调整原因（建议填写）
7. 点击「确定」提交

**系统自动完成：**
- ✅ 生成库存调整流水记录
- ✅ 调整单号自动生成（格式：`IA + 日期 + 序号`）

> 💡 **首次使用提示：** 系统上线时可以通过「库存调整 - 增加」来录入初始库存。

---

### 5.4 库存盘点

**路径：** 库存管理 → 盘点功能

定期对仓库进行实物盘点，核对系统库存与实际库存是否一致。

**盘点类型：**
| 类型 | 说明 |
|------|------|
| 全盘 (full) | 对仓库所有商品进行盘点 |
| 部分盘点 (partial) | 对部分商品进行盘点 |
| 循环盘点 (cycle) | 按计划轮流盘点不同商品 |

**操作步骤：**

**第一步：创建盘点单**
1. 点击「新增盘点单」
2. 选择仓库
3. 选择盘点类型（全盘 / 部分 / 循环）
4. 选择盘点日期
5. 点击「确定」保存

**第二步：录入盘点明细**
1. 在盘点单列表中点击该行的「明细」按钮，打开盘点明细弹窗
2. 使用**扫码添加商品**输入框（支持 PDA 扫码枪）逐件扫码，或点击「添加行」手动添加商品
   - 扫码时：每扫一次自动添加该商品，若已存在则实盘数量 +1（适合边扫边点）
3. 系统自动填充**账面数量**（系统当前库存）
4. 手动输入或通过扫码累加**实盘数量**（实际清点数量）
5. 系统自动计算**差异数量** = 实盘数量 - 账面数量
6. 点击「保存」保存明细

**第三步：完成盘点**
1. 确认所有明细已录入并保存
2. 在该盘点单行点击「完成」按钮
3. 系统自动完成：
   - ✅ 生成库存调整单（针对有差异的商品）
   - ✅ 生成库存调整流水记录
   - ✅ 库存自动调整为实盘数量

> ⚠️ **重要：** 盘点完成后不可撤销，请确认数据无误后再操作。

| 作废 (void) | 作废流水 |
| 付款 (pay) | 付款操作 |
| 收款 (collect) | 收款操作 |

**查看方式：**

- 点击某条记录的「详情」按钮，在弹窗中查看**变更详情**（含变更前后字段对比）

> 💡 **提示：** 每个企业只能查看本企业的操作日志，数据完全隔离。

## 十、PDA 扫码功能

系统支持 PDA 扫码枪或手动输入条码/编码，在部分业务页面快速选择商品，提高录入效率。

### 10.1 前置条件

1. **商品需维护条形码或编码**：在「商品管理」中为商品填写**条形码**（或使用商品编码）
2. **PDA 设备**：使用支持 HID 键盘模式的扫码枪，扫码后会将条码内容输入到当前焦点输入框并发送回车

### 10.2 支持扫码的页面

| 页面 | 使用方式 |
|------|----------|
| 库存盘点 → 盘点明细 | 扫码添加商品；同一商品多次扫码，实盘数量自动累加 |
| 库存调整 | 商品选择旁扫码，自动填入商品 |
| 库存调拨 | 商品选择旁扫码，自动填入商品 |
| 入库记录 | 新增入库单时，商品选择旁扫码 |
| 出库记录 | 新增出库单时，商品选择旁扫码 |
| 采购订单 → 采购明细 | 扫码添加商品；同一商品多次扫码，数量自动累加 |
| 销售订单 → 销售明细 | 扫码添加商品；同一商品多次扫码，数量自动累加 |

### 10.3 使用方式

- **PDA 扫码枪**：光标置于扫码输入框 → 使用扫码枪扫描商品条码 → 系统自动识别并添加/选中商品
- **手动输入**：在扫码输入框中输入商品**条码**或**编码** → 按回车 → 系统查询并添加/选中商品
- **手机扫码 App**：通过手机扫码后复制条码，粘贴到输入框并按回车

> 💡 **提示：** 系统优先按条形码查询，未找到时再按商品编码查询，支持两种方式混用。

## 十一、常见业务场景操作指南

### 场景 1：完整的采购入库到付款流程

```
1. 【采购管理 → 采购订单】创建采购订单，选择供应商和商品
2. 【采购管理 → 采购订单】到货后点击「收货」
   → 系统自动：库存增加 + 生成应付账款
3. 【财务管理 → 应付账款】找到对应应付记录，点击「付款」
   → 输入付款金额，完成付款
```

### 场景 2：完整的销售出库到收款流程

```
1. 【销售管理 → 销售订单】创建销售订单，选择客户和商品
2. 【销售管理 → 销售订单】确认后点击「发货」
   → 系统自动：库存减少 + 生成应收账款
3. 【财务管理 → 应收账款】找到对应应收记录，点击「收款」
   → 输入收款金额，完成收款
```

### 场景 3：客户退货退款

```
1. 【销售管理 → 退货处理】点击「新增退货单」，创建退货单并关联原销售订单
2. 【销售管理 → 退货处理】点击该退货单的「处理入库」按钮
   → 系统自动：商品退回入库 + 冲减应收账款
3. 【销售管理 → 退货处理】如需退款，点击该退货单的「一键退款」或「退款闭环」按钮
   → 系统自动生成退款流水
```

### 场景 4：客户换货

```
1. 【销售管理 → 换货管理】点击「新增换货单」，创建换货单并关联原销售订单
2. 【销售管理 → 换货管理】在该换货单行点击「完成」按钮
   → 系统自动：退回商品入库 + 换出商品出库
```

### 场景 5：月底盘点

```
1. 【库存管理 → 盘点功能】点击「新增盘点单」，选择仓库、盘点类型和盘点日期并保存
2. 【库存管理 → 盘点功能】在该盘点单行点击「明细」，在弹窗中可使用扫码添加商品（每扫一件实盘数量+1），或手动添加行并录入实盘数量，保存明细
3. 【库存管理 → 盘点功能】确认无误后点击该盘点单的「完成」按钮
   → 系统自动：计算差异并生成调整记录，库存自动修正
```

### 场景 6：仓库间调拨

```
1. 【库存管理 → 调拨管理】点击「新增调拨单」
2. 选择商品、调出仓库和调入仓库，填写调拨数量和单位成本
3. 点击「确定」提交
   → 系统自动：调出仓库减少库存 + 调入仓库增加库存
```

### 场景 7：初始库存录入（系统上线时）

```
1. 【库存管理 → 库存调整】点击「新增调整」
2. 选择商品、仓库，类型选择「增加」
3. 填写调整数量和单位成本（及调整原因），点击「确定」提交
   → 系统自动生成入库记录，建立初始库存
```

### 场景 8：作废一笔错误的付款

```
1. 【财务管理 → 收支明细】找到错误的付款记录
2. 点击「作废」
   → 系统自动：回滚对应应付账款的已付金额
3. 重新在【财务管理 → 应付账款】进行正确的付款
```

## 十二、常见问题（FAQ）

### Q1：忘记密码怎么办？
可点击登录页的「忘记密码？」链接，输入注册邮箱后收取重置链接并设置新密码（需管理员已开启找回密码功能）。若未开启或无法收到邮件，请联系企业管理员，管理员可在「用户管理」中重置您的密码。

### Q2：为什么我看不到某些菜单？
系统根据您的角色权限显示菜单。如需更多权限，请联系管理员在「角色配置」中调整。

### Q3：发货时提示"库存不足"？
请检查对应仓库的库存是否充足。可以在「实时库存」页面查看当前库存，按仓库筛选确认。如果是新商品，需先通过采购入库或库存调整来增加库存。

### Q4：如何修改已收货的采购订单？
已收货的采购订单不能直接修改。需要先「取消」该订单（系统会回滚库存和应付），然后重新创建正确的采购订单。注意：如果已有付款记录，需先作废相关付款流水。

### Q5：应收/应付状态为什么显示"逾期"？
当超过到期日期仍有未收/未付余额时，状态会自动显示为「逾期」。请及时跟进收付款。

### Q6：支持多个仓库吗？
支持。您可以在「仓库管理」中创建多个仓库，采购入库和销售出库时选择对应的仓库，库存数据按仓库独立管理。也可以通过「调拨管理」在仓库间调拨库存。

### Q7：删除的数据能恢复吗？
系统采用软删除机制，已删除的数据在数据库中保留。如需恢复，请联系系统管理员。

### Q8：如何查看某个操作是谁做的？
在「操作日志」中可以查看所有操作记录，包括操作人、操作时间、操作类型和变更详情。

### Q9：一个企业可以创建多少用户？
用户数量取决于您的套餐：

| 套餐 | 最大用户数 |
|------|-----------|
| 免费版 (free) | 有限制 |
| 基础版 (basic) | 根据套餐 |
| 专业版 (pro) | 根据套餐 |
| 企业版 (enterprise) | 根据套餐 |

具体数量请查看您的套餐详情或联系平台管理员。

### Q10：数据安全吗？不同企业的数据会混淆吗？
系统采用严格的数据隔离机制，每个企业的数据完全独立。您只能看到自己企业的数据，其他企业无法访问您的任何信息。

### Q11：PDA 扫码没有反应怎么办？
请确认：① 商品已在「商品管理」中填写条形码或编码；② 扫码枪处于 HID 键盘模式（扫码后模拟键盘输入+回车）；③ 光标已聚焦在扫码输入框内。也可在输入框中手动输入条码/编码后按回车进行测试。
