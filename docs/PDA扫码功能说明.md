# PDA 扫码功能说明

> 版本：v1.0 | 更新日期：2026年2月14日

---

## 一、产品分析：适合 PDA 扫码的场景

从进销存业务场景出发，以下模块**最适合**引入 PDA 扫码：

| 优先级 | 模块 | 场景说明 | 扫码价值 |
|--------|------|----------|----------|
| **高** | **库存盘点** | 仓管员手持 PDA 在库区逐件扫码，系统自动添加商品并累加实盘数量 | 盘点效率显著提升，减少手工选品和录入错误 |
| **高** | **采购收货** | 供应商送货时，收货员扫码核对商品和数量 | 快速验货、减少错收漏收 |
| **高** | **销售发货** | 拣货/发货时扫码确认出库商品 | 提高拣货准确率、减少发错货 |
| **中** | **采购订单 / 销售订单** | 录入明细时扫码添加商品，可自动累加数量 | 订单录入更快，适合大批量商品 |
| **中** | **库存调拨** | 调拨时扫码选择商品 | 避免选错商品 |
| **中** | **库存调整** | 手动调整时扫码选择商品 | 便于快速定位商品 |
| **中** | **入库记录 / 出库记录** | 手工录入出入库时扫码选品 | 提升录入效率 |

## 二、已实现功能

### 2.1 后端 API

- **条码/编码查询接口**：`GET /api/v1/products/lookup?barcode=xxx` 或 `?code=xxx`
  - 支持按条形码或商品编码精确查询
  - 用于 PDA 扫码枪输入后快速匹配商品
- **商品列表搜索增强**：商品列表的 `search` 参数现已支持按条形码模糊搜索

### 2.2 可复用组件

- **BarcodeScanInput**：`resources/js/components/BarcodeScanInput.vue`
  - 输入框 + 查询按钮
  - 支持 PDA 扫码枪（扫码枪模拟键盘输入，扫码后自动发送回车）
  - 支持手动输入条码/编码后按回车查询
  - 支持条码、编码两种查询方式（先查条码，未找到则查编码）

### 2.3 已集成页面

| 页面 | 集成方式 |
|------|----------|
| **库存盘点 → 盘点明细** | 扫码添加商品；已存在则实盘数量 +1 |
| **库存调整** | 商品选择旁增加扫码输入框 |
| **库存调拨** | 商品选择旁增加扫码输入框 |
| **入库记录** | 商品选择旁增加扫码输入框 |
| **出库记录** | 商品选择旁增加扫码输入框 |
| **采购订单 → 采购明细** | 扫码添加商品；已存在则数量 +1 |
| **销售订单 → 销售明细** | 扫码添加商品；已存在则数量 +1 |

## 三、使用说明

### 3.1 前置条件

1. **商品需维护条形码**：在「商品管理」中为商品填写条形码字段（或使用商品编码）
2. **PDA 设备**：使用支持 HID 键盘模式的扫码枪 / PDA，扫码后会将条码内容输入到当前焦点输入框并发送回车

### 3.2 使用方式

- **盘点场景**：打开盘点明细弹窗 → 光标置于扫码输入框 → 使用 PDA 逐个扫码商品 → 每扫一次自动添加/累加
- **订单/调整/调拨/出入库**：在对应表单中，使用商品旁的扫码框扫码，或手动输入条码/编码后按回车

### 3.3 兼容性

- 支持 PDA 扫码枪（HID 键盘模式）
- 支持手机扫码 App + 手动粘贴条码
- 支持直接输入商品编码查询

## 四、技术说明

### 4.1 扫码枪工作原理

多数 PDA 扫码枪以 USB HID 键盘形式工作：扫码后向系统发送一串按键序列，最后发送 Enter。因此只需在输入框获得焦点时扫码，系统即可收到条码字符串并触发查询。

### 4.2 测试

**后端 API 测试**（不依赖浏览器）：
```bash
php artisan test --filter=ProductLookupTest
```

**E2E 模拟扫码测试**（需先安装 Playwright 浏览器）：
```bash
npx playwright install
php artisan serve &
npm run e2e -- e2e/barcode-scan.spec.cjs
```

测试覆盖：盘点明细、库存调整、调拨、入库、出库、采购订单、销售订单、按编码查询。

---

### 4.3 扩展建议

- **采购收货 / 销售发货**：若系统有独立的收货/发货录入页，可参照入库记录、出库记录的方式集成扫码
- **移动端**：若需在手机端扫码，可引入 `html5-qrcode` 等库实现摄像头扫码，再调用 `products/lookup` 接口
