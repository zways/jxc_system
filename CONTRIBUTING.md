# 贡献指南

感谢你愿意为本项目贡献代码。

## 开始之前

- 先阅读 `README.md`，确认本地环境可正常运行。
- 提交前请确保不包含敏感信息（如 `.env`、测试账号、本地数据导出）。
- 大改动建议先提 Issue 讨论，再开始开发。

## 开发流程

1. Fork 本仓库并创建分支：
   - `feature/xxx`：新功能
   - `fix/xxx`：修复
   - `docs/xxx`：文档
2. 本地开发并保持提交粒度清晰。
3. 提交前自检：
   - 后端测试：`php artisan test`
   - 前端构建：`npm run build`
4. 发起 Pull Request 并填写模板。

## 提交规范（建议）

- `feat:` 新功能
- `fix:` 修复问题
- `docs:` 文档更新
- `refactor:` 重构
- `chore:` 构建/配置/杂项

示例：`feat: add tenant inventory summary api`

## 代码风格

- PHP：遵循 Laravel 社区习惯，建议使用 Pint 格式化。
- 前端：保持组件职责单一，命名语义清晰。
- 避免提交无关改动（如大面积格式化、临时调试代码）。

## Pull Request 要求

- 说明变更目的、主要改动点和影响范围。
- 提供必要的测试说明（如何复现、如何验证）。
- 如涉及 UI 变更，附截图或录屏。

再次感谢你的贡献。
